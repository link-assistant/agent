# Case Study: Issue #247 — Rust CI/CD Releases Not Working

## Summary

The Rust CI/CD pipeline never produces GitHub releases despite successful builds and tests on pushes to `main`. The JS pipeline works correctly because it uses different conditional patterns.

## Timeline / Sequence of Events

1. **Initial Setup**: Rust CI/CD pipeline (`rust.yml`) was created with `auto-release` and `manual-release` jobs modeled after the JS pipeline but with key differences in job conditions.
2. **Changelog fragments accumulated**: Two changelog fragments exist in `rust/changelog.d/`:
   - `20251230_120000_separate_cicd_pipelines.md` (minor bump)
   - `20260410_201700_add_temperature_option.md` (minor bump)
3. **Multiple pushes to main**: Commits like `fb29e42` (2026-04-11) and `a91b6c3` (2026-04-10) triggered the Rust CI/CD pipeline.
4. **All CI jobs pass**: `lint`, `test`, `build` all succeed.
5. **Auto Release always skipped**: In every run, the `auto-release` job status is "skipped" with zero steps executed.
6. **No Rust releases exist**: `gh release list` shows only JS releases (`[js] 0.22.0`, etc.). No `rust-v*` tags exist.

## Root Causes

### Root Cause 1 (Critical): Missing `always()` in `auto-release` Job Condition

**Location**: `.github/workflows/rust.yml`, line 185

**The bug**: The `auto-release` job condition is:
```yaml
if: github.event_name == 'push' && github.ref == 'refs/heads/main'
```

**Why it fails**: The `changelog-check` job (line 46) has `if: github.event_name == 'pull_request'`, so it is **skipped** on push events. In GitHub Actions, when any job in the transitive dependency chain is skipped, all downstream jobs without `always()` are also skipped by default — even if the immediate `needs` jobs all succeed.

The dependency chain is:
```
changelog-check (SKIPPED on push) → lint (uses always(), PASSES) → build (uses always(), PASSES)
changelog-check (SKIPPED on push) → test (uses always(), PASSES) → build (uses always(), PASSES)
lint, test, build → auto-release (NO always(), SKIPPED)
```

**Contrast with JS pipeline**: The JS `release` job (`.github/workflows/js.yml`, line 196) correctly uses:
```yaml
if: always() && github.ref == 'refs/heads/main' && github.event_name == 'push' && needs.lint.result == 'success' && needs.test.result == 'success' && needs.verbose-integration.result == 'success'
```

**Fix**: Add `always()` and explicit result checks to the `auto-release` condition.

### Root Cause 2 (Secondary): Changelog Version Regex Mismatch in `create-github-release.mjs`

**Location**: `scripts/create-github-release.mjs`, line 82

**The bug**: The regex used to extract release notes from CHANGELOG.md is:
```js
const versionHeaderRegex = new RegExp(`## ${version}[\\s\\S]*?(?=## \\d|$)`);
```

This searches for `## 0.8.0` but the Rust changelog format (generated by `rust-version-and-commit.mjs` line 201 and `rust-collect-changelog.mjs` line 107) produces:
```markdown
## [0.8.0] - 2026-04-11
```

The `[` character would be interpreted as a regex character class start, and the match would fail.

**Contrast with JS**: JS uses `## 0.22.0` format (without brackets), which matches the regex correctly.

**Fix**: Update the regex to handle both `## version` and `## [version] - date` formats.

### Root Cause 3 (Minor): Missing `format-github-release.mjs` Step in Rust Workflow

**Location**: `.github/workflows/rust.yml`, `auto-release` job

**The bug**: The JS `release` job has a "Format GitHub release notes" step that calls `format-github-release.mjs` to enhance the release with PR links and badges. The Rust `auto-release` job is missing this step.

**Fix**: Add the `format-github-release.mjs` step to the Rust `auto-release` and `manual-release` jobs.

### Root Cause 4 (Minor): Same Missing `always()` in `manual-release` Job

**Location**: `.github/workflows/rust.yml`, line 266

**The bug**: The `manual-release` job has the same problem — it lacks `always()` and explicit result checks in its `if` condition:
```yaml
if: github.event_name == 'workflow_dispatch'
```

**Fix**: Add `always()` and explicit result checks.

## Solutions

### Solution 1: Fix `auto-release` Job Condition (Primary Fix)

Change the `if` condition from:
```yaml
if: github.event_name == 'push' && github.ref == 'refs/heads/main'
```
to:
```yaml
if: always() && github.event_name == 'push' && github.ref == 'refs/heads/main' && needs.lint.result == 'success' && needs.test.result == 'success' && needs.build.result == 'success'
```

### Solution 2: Fix Changelog Regex in `create-github-release.mjs`

Update the regex to handle both plain and bracketed version formats:
```js
const versionHeaderRegex = new RegExp(`## \\[?${version.replace(/\./g, '\\.')}\\]?[\\s\\S]*?(?=## [\\[\\d]|$)`);
```

### Solution 3: Add `format-github-release.mjs` Step to Rust Workflow

Add the missing formatting step after the GitHub release creation step in both `auto-release` and `manual-release` jobs.

### Solution 4: Fix `manual-release` Job Condition

Apply the same `always()` pattern to the `manual-release` job.
